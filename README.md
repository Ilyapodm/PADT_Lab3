## Архитектура
### Принцип подстановки Лисков (LSP)
`Дочерний класс должен дополнять, а не замещать поведение родительского класса.`

Если дочерний класс подставить в места использования базового класса, ничего поменяться не должно.
LSP говорит о поведении объектов после создания (`SquareMatrix` валидируется только при создании, потом работает как `Matrix`).

### DynamicArray\<T\>
#### Метод resize() в DynamicArray
Работа с размером массива происходит всегда через этот метод.
`resize()` увеличивает size на заданное количество и не меняет `capacity`, если `size + new_size <= capacity`. Если же `capacity` не хватает для всех `size + new_size` - то выделяется 2x памяти от исходного `capacity`, и туда копируются данные.
PS. Если не хватит 2x `capacity`, то выделиться `size + new_size` ячеек.

### Complex\<T\>

Функция `magnitude()` для `Complex<T>` и `int`, `double` (используется в методе `norm()`) вынесена в отдельный файл math_type.hpp, чтобы  ADL (Argument-Dependent Lookup) правильно выбрал реализацию без лишних include'ов

### AlgebraTraits<T>
Единая точка алгебраических примитивов: `zero()`, `one()`, 
`magnitude()`, `random_value()`. Частичная специализация для `Complex<T>`.
Мотивация: частичная специализация функций запрещена в C++ — поэтому struct.

### IMatrix\<T\>

#### Требования
Тип `T` должен иметь:
- Конструктор `T{}`, при том `T{}` должен является аддитивным нулем (нейтральным по сложению)
- Конструктор копирования (используется в `get()`, и `set()`)
- Перегруженные `operator+` и `operator*`
- Перегруженную функцию `magnitude(T x)` (используется в `norm()`)


#### Схема
- Dense - обычное хранение матрицы.
- Sparse - разреженное хранение (только ненулевые элементы).

```
IMatrix<T>             ← интерфейс
    ├── Matrix<T>      ← обычная прямоугольная (DynamicArray плоский)
    │       ├── SquareMatrix<T>   ← квадратная (rows==cols)
    │       └── Vector<T>         ← обертка для SystemOfEquations (вектор-столбец b)
    ├── TriangularMatrix<T>       ← своё хранение n*(n+1)/2
    ├── DiagonalMatrix<T>         ← хранит только диагональ DynamicArray(n)
    ├── SparseMatrix<T>           ← DynamicArray<Triplet>
```

Оператор `(i, j)` только для чтения реализован в самом интерфейсе и доступен всем потомкам при переопределении `get(i, j)`.

#### Детали реализации

Методы `add()` и `mult_scalar()` используют ковариантные возвращаемые типы: каждый класс возвращает наиболее конкретный гарантированный тип (например, `SquareMatrix*` из `TriangularMatrix::add()`).

Метод `in_bounds()` не был вынесен в `IMatrix`, так как он не сложный (всего пару строк, можно и написать несколько раз), а при его использовании в таком случае дергались бы 2 виртуальных метода - overhead.

Также в `IMatrix` не был вынесен `T& operator()`, так как у каждого разные ограничения на set.

#### TriangularMatrix
Для нижнетреугольной матрицы используем Raw-Major: Matrix(i, j) = data[i*(i+1)/2 + j]
Для верхнетреугольной матрицы используем Column-Major: Matrix(i, j) = data[j*(j+1)/2 + i]

Есть один ноль на всю матрицу, который существует все время существования программы: `inline static const T ZERO{}`

#### SparseMatrix
Есть один ноль на всю матрицу, который существует все время существования программы: `inline static const T ZERO{}`

### SystemOfEquations
Решение системы линейных алгебраических уравнений разными методами со сравнением времени работы этих методов в разных условиях. 

Методы решения не были вынесены в отдельные классы, так как условие ЛР по ЛАВМ предполагает использование только 3-х методов, а для 3-х методов это было бы слишком долго, а расширяемости, ради которой это следует делать, нет.

#### Сеттеры
| Ситуация                         | Метод         |
| -------------------------------- | ------------- |
| Меняется только A, размер тот же | set_matrix(A) |
| Меняется только b, размер тот же | set_rhs(b)    |
| Меняется размер / оба сразу      | set(A, b)     |

#### solve_gauss()
Решение СЛАУ стандартным методом Гаусса

#### solve_gauss_with_pivot()
Решение СЛАУ методом Гаусса с выбором ведущего элемента. 

Просто для каждого столбца выбираем максимальный элемент (ниже диагонального или равный ему) и переставляем строки, таким образом избегая деление на маленькие по модулю числа, которые вносят неточность.

#### hilbert(int n)
Для проверки точности методов используется матрица Гильберта:

$$H_{ij} = \frac{1}{i+j-1}, \quad i,j \in \{1, \ldots, n\}$$

Система конструируется так, что точное решение известно заранее:
задаётся $x^* = (1, 1, \ldots, 1)^T$, правая часть вычисляется как $b = Hx^*$.
Это позволяет измерить относительную погрешность $\|{\tilde{x} - x^*}\| / \|{x^*}\|$
