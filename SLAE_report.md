## 2. Описание алгоритмов

### 2.1 Метод Гаусса без выбора ведущего элемента

Прямое устранение: для каждого столбца k вычисляется множитель `scale = A[i][k] / A[k][k]` и строка i вычитается из всех нижележащих строк. Затем обратная подстановка. Сложность O(n³). При нулевом пивоте — исключение.

### 2.2 Метод Гаусса с выбором ведущего элемента (частичный выбор по столбцу)

Перед каждым шагом устранения ищется строка с максимальным по модулю элементом в текущем столбце и переставляется на место пивота. Снижает накопление ошибок округления и уменьшает вероятность деления на малое число.

### 2.3 PLU-разложение

Матрица A разлагается в произведение PA = LU, где P — матрица перестановок, L — нижнетреугольная с единицами на диагонали (алгоритм Дулиттла), U — верхнетреугольная. Разложение вычисляется один раз за O(n³) и кэшируется. Каждое последующее решение с новой правой частью требует только O(n²).

### 2.4 Прямая и обратная подстановки

После разложения задача сводится к двум треугольным системам:

    Ly = Pb — прямая подстановка сверху вниз, y[i] = Pb[i] − Σ L[i][j]·y[j]

    Ux = y — обратная подстановка снизу вверх, x[i] = (y[i] − Σ U[i][j]·x[j]) / U[i][i]

## 5. Выводы

### 5.1 Сравнительная эффективность методов по времени

Все три метода имеют одинаковую асимптотику O(n³) и показывают сравнимое время при решении одной системы. Время подстановки PLU пренебрежимо мало (10 мс против 7200 мс разложения при n=1000), что делает повторное использование разложения крайне выгодным.

### 5.2 Влияние выбора ведущего элемента на точность

Для n=5 все методы дают точность на уровне машинного эпсилона (~10⁻¹³), разница незначима. Для n=10 Гаусс с выбором и PLU точнее обычного Гаусса примерно в 1.5 раза по относительной погрешности. Для n=15 число обусловленности матрицы Гильберта (~10¹⁷) превышает обратное машинного эпсилона (~10¹⁶): все методы дают погрешность порядка единиц при невязке ~10⁻¹⁶. Малая невязка при огромной погрешности — прямое следствие плохой обусловленности, не ошибки алгоритма.
### 5.3 Преимущество LU-разложения при нескольких правых частях

При k=1 выигрыша нет. При k=100 LU решает в 76 раз быстрее Гаусса с выбором (1183 мс против 90499 мс). Причина: LU выполняет дорогое разложение O(n³) один раз, а каждую из k правых частей обрабатывает за O(n²). Гаусс вынужден каждый раз повторять O(n³). Выигрыш растёт линейно с k.
